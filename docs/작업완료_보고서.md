# MONO Phase 1~4 작업완료 보고서

## 1. 개요
- 프로젝트: MONO (PWA 기반 실시간 번역 메신저)
- 목표: 현장 운용 가능한 안정성 + 메신저 기능 확장 + 수익화/운영 기반 구축
- 기간 범위: Phase 1 ~ Phase 4

## 2. Phase별 완료 내역

## Phase 1: 기반 메신저화
- 하단 탭/메인 네비게이션 구조 정리 및 주요 화면 접근성 개선
- 연락처 탭 구조 개선(친구/초대/검색 기반 흐름)
- 1:1 대화방 버블/입력/상태 UI 정리
- 로컬 대화 저장소를 IndexedDB 중심으로 전환
- 서버 기본 API 정리:
  - 사용자/친구/방/멤버 조회 및 읽음 상태 갱신
- DB 마이그레이션 기반 스키마 정립(users, friends, rooms, room_members)

## Phase 2: 메신저 완성도 강화
- 메시지 ACK/재시도/중복방지 안정화
- 읽음/전달 상태 이벤트 정합성 강화
- 오프라인 큐(outbox) 및 재연결 flush 신뢰성 보강
- 소켓 이벤트 스키마 일관화(`text/originalText/translatedText/status`)
- 회귀 자동검증(qa-smoke + stress 시나리오) 체계화
- 설정 화면 비로그인/로그인 분기 정상화 및 Google 로그인 연동

## Phase 3: 글로벌/특화 기능
- Global 탭 최소 실동작 구현(입장/상태/오류 처리)
- 1:1/글로벌 공통 송수신 흐름 정합성 강화
- 운영 전 문서/체크리스트(스모크/검증 명령) 정리

## Phase 4: 고도화 + 수익화

### 4-1 과금 기반
- DB 확장:
  - `users.plan`, `users.plan_expires_at`
  - `translation_usage(user_id, month, count, updated_at)`
- 서버 과금 로직:
  - 번역 사용량 월간 집계
  - free 한도 초과 체크(`checkUsageLimit` 구조 포함)
  - 한도 초과 시 사용자 경고 이벤트 전송
- API:
  - `GET /api/subscription/me`
  - `GET /api/subscription/check-limit`
  - `POST /api/subscription/checkout` (결제 연동 스텁)
  - `POST /api/subscription/webhook` (웹훅 스텁)
- 클라이언트:
  - 설정 > 구독관리 카드
  - 이번 달 번역 횟수/플랜 표시 및 업그레이드 버튼

### 4-2 소셜로그인 확장
- OAuth provider 컬럼:
  - `google_id`, `kakao_id`, `line_id`, `apple_id`
- 계정 병합:
  - 동일 이메일 시 기존 계정 연결(통합)
  - JWT `sub`를 병합된 실제 user id 기준으로 처리
- 라우트:
  - Google: `/auth/google`, `/api/auth/google`
  - Kakao: `/auth/kakao`, `/api/auth/kakao`
  - LINE: `/auth/line`, `/api/auth/line`
  - Apple: `/auth/apple`, `/api/auth/apple`
  - callback alias 포함
- 설정 비로그인 화면:
  - Google/Kakao/LINE/Apple 로그인 버튼 + 브랜드 스타일 적용

### 4-3 폰트 + 디자인 시스템
- 폰트 적용:
  - Pretendard, Inter, Noto Sans JP
- 디자인 토큰 도입:
  - `--color-primary`, `--color-bg`, `--color-text`
  - `--color-bubble-mine`, `--color-bubble-other`, `--color-border`
  - spacing/radius 토큰
- `.dark` 오버라이드 추가
- 기존 `--mono-*`와 호환 매핑 유지(기존 UI 깨짐 방지)
