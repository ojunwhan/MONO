### 1.3 `/hospital` 쿼리 파라미터 (HospitalApp)

- `template`: `reception` | `consultation` — 접수처 vs 상담실
- `room`: 방 ID (직원 PC/키오스크 구분 시 필수)
- `kiosk`: `true` → 키오스크 QR 전용 화면 (상담실이면 QR만 표시, 환자 도착 시 `/fixed-room/:roomId` 전환)
- `org`: (선택) 조직 ID
- **동작:**
  - `template` + `room` 있고 `kiosk` 없음 → 직원 PC 통역 화면
  - `template` + `room` 있고 `kiosk=true` → 키오스크 QR 화면 (상담실이면 `hospital:consultation:watch` + `hospital:patient-arrived` 수신 시 `/fixed-room/:roomId` 이동)
  - 위 조합 아니면 → `/hospital-dashboard` 리다이렉트 (HospitalApp 내부 `shouldRedirect`)

## 3. 통역 파이프라인

**관련 파일:**  
- 클라이언트: `src/hooks/useVADPipeline.js`, `src/components/MicButton.jsx`, `src/pages/FixedRoomVAD.jsx` (PTT: MediaRecorder → PCM16 전송)  
- 서버: `server.js` (stt:open, stt:audio, stt:segment_end, transcribePcm16, fastTranslate, hqTranslate)

### 3.1 전체 흐름

1. **음성 입력**
   - **VAD 모드:** Silero VAD (useMicVAD)로 발화 종료 감지 → Float32 → Int16 PCM 16kHz → base64 청크로 `stt:audio` 연속 전송 후 `stt:segment_end`
   - **PTT 모드 (FixedRoomVAD):** MediaRecorder로 녹음 → stop 시 Blob → decodeAudioData → 16kHz 리샘플 → Int16 PCM → `stt:open` + `stt:audio` 청크 + `stt:segment_end` (VAD/SharedArrayBuffer 미사용)
   - **MicButton (일반 채팅):** 녹음 버튼 누르면 MediaRecorder/스트림 → `stt:open` / `stt:audio` / `stt:segment_end` 동일 프로토콜

2. **서버**
   - `stt:open`: STT_SESSIONS에 세션 등록 (roomId, participantId, lang, sampleRateHz)
   - `stt:audio`: 청크 누적 (session.chunks), 크기/rate limit
   - `stt:segment_end`: 누적 PCM 합침 → `transcribePcm16()` → (짧은 구간/저음량 필터) → `fastTranslate()` (스트리밍 옵션) → `receive-message-stream` / `receive-message-stream-end` 발송 → 이후 `hqTranslate()`로 HQ 번역 → `revise-message` 발송

3. **출력**
   - 클라이언트: `receive-message-stream`으로 실시간 번역 표시, `receive-message-stream-end`로 최종문 반영, `receive-message`로 메시지 추가, `revise-message`로 HQ로 갱신

### 3.2 VAD 로직 (Silero VAD)

- **라이브러리:** `@ricky0123/vad-react` (useMicVAD)
- **설정:** `startOnLoad: false`, baseAssetPath/onnxWASMBasePath: `/`, positiveSpeechThreshold 0.5, negativeSpeechThreshold 0.35, redemptionMs 600, minSpeechMs 250, preSpeechPadMs 300, submitUserSpeechOnPause: true
- **발화 종료 시:** onSpeechEnd(audioFloat32) → RMS 저음량 필터(vadThresholdRef) → 최소 길이 필터(minSpeechSamplesRef) → sendAudioToServer(audioFloat32)
- **shortFragment:** 서버 측에서 duration < STT_MIN_DURATION_SEC 또는 rms < STT_MIN_RMS 이면 스킵 후 `stt:no-voice` 가능

### 3.3 STT: Groq Whisper Large V3

- **함수:** `transcribePcm16(pcmBuffer, lang, sampleRateHz, opts)`
- PCM16 → WAV 버퍼 → 임시 파일 → Groq `groq.audio.transcriptions.create({ file, model: "whisper-large-v3", response_format: "json", temperature: 0, language })`
- **병원 모드:** `opts.hospitalMode && groq` 이면 Groq만 사용 (fallback 없음). 그 외에는 Groq 우선, 없으면 OpenAI whisper-1 fallback
- **환각 필터:** `isWhisperHallucinationText(text)`, `isGarbageText(text)` 등으로 스킵 가능

### 3.4 번역: GPT-4o / buildSystemPrompt

- **fastTranslate:** 스트리밍 번역용 (onChunk 콜백으로 `receive-message-stream` 발송)
- **hqTranslate:** 최종 고품질 번역 → `revise-message` / `tts_audio` 등
- **buildSystemPrompt(from, to, ctx, siteContext, opts):**
  - `siteContext`가 `hospital_` 로 시작하면 **병원 경어체** 지시문 추가 (CRITICAL — Hospital Mode Language Register: 최고 존칭, ~하십니까, Would you please~ 등)
  - 의료 용어 컨텍스트: `getMedicalTermContext(dept, to)` 사용
  - 일반 모드: 캐주얼 메신저 톤, 슬랭/존댓말 유지 등

### 3.5 스트리밍

- 서버: `fastTranslate(..., { stream: true, onChunk })` → 수신자 소켓으로 `receive-message-stream` (messageId, chunk, originalText 등) → 완료 시 `receive-message-stream-end` (fullText 등)
- 클라이언트: ChatScreen / FixedRoomVAD 등에서 해당 이벤트 수신 후 말풍선에 청크 누적 표시, stream-end 시 최종문 반영

### 3.6 PTT vs VAD

- **VAD:** useVADPipeline + useMicVAD 사용. 발화 자동 감지 → 종료 시 한 번에 전송. FixedRoomVAD에서 inputMode===`vad`일 때만 사용 (InterpretingVAD).
- **PTT:** FixedRoomVAD에서 inputMode===`ptt`일 때 VAD 미사용. MediaRecorder로 버튼 누른 동안만 녹음 → 버튼 해제 시 PCM16 변환 후 동일하게 `stt:open` / `stt:audio` / `stt:segment_end` 전송.

## 4. 병원 모드 전체 로직

**관련 파일:**  
`src/pages/HospitalApp.jsx`, `src/pages/HospitalPatientJoin.jsx`, `src/pages/HospitalDashboard.jsx`, `src/pages/FixedRoomVAD.jsx`, `server.js` (POST /api/hospital/join, hospital_rooms, hospital_sessions, 소켓 hospital:*)

### 4.1 접수처(PTT) 플로우

- **직원 PC:** `/hospital?template=reception&room={roomId}` → HospitalApp에서 통역 화면 (대기실 목록, 환자 픽 등)
- **환자:** QR 스캔 → `/hospital/join/reception` (HospitalPatientJoin) → 언어 선택 → "통역 시작" → POST `/api/hospital/join` (department: reception) → roomId 수신 → `/room/:roomId` (ChatScreen, PTT)
- 직원 ↔ 환자 1:1 채팅/통역 (PTT 버튼으로 녹음)

### 4.2 상담실(VAD) 플로우

- **태블릿(키오스크):** `/hospital?template=consultation&room={roomId}&kiosk=true` → ConsultationKioskView → QR 표시 (URL: `/hospital/join/consultation?room={roomId}`)
- **환자:** QR 스캔 → `/hospital/join/consultation?room={roomId}` → **언어 선택 화면** 표시 (자동 join 제거됨) → 언어 선택 후 "통역 시작" → POST `/api/hospital/join` (department: consultation, room: consultationRoomId) → roomId (PT-XXXXXX) + sessionId 수신 → `/fixed-room/:roomId` (FixedRoomVAD, state에 inputMode: "vad", hospitalTemplate: "consultation")
- **태블릿:** socket `hospital:consultation:watch` → 환자 join 시 서버가 `hospital:patient-arrived` emit → payload.roomId로 `/fixed-room/:roomId` 자동 전환 (state: inputMode "vad", roleHint "guest" 등)
- **통역:** FixedRoomVAD에서 직원이 "통역 시작" → `fixed-room:start` → 양쪽 VAD 시작 (또는 PTT 선택 시 MediaRecorder)

### 4.3 /hospital 분기 (HospitalApp)

- `template=reception` + `room` + `!kiosk` → 직원 PC 접수 화면
- `template=consultation` + `room` + `kiosk=true` → 상담실 키오스크 QR 화면 (환자 도착 시 `/fixed-room/:roomId` 전환)
- 그 외 `template`+`room` 조합 없거나 kiosk만 있으면 → `/hospital-dashboard` 리다이렉트

### 4.4 kiosk=true 동작

- 상담실: QR만 상시 표시, `hospital:consultation:watch` 구독, `hospital:patient-arrived` 수신 시 같은 소켓에서 `/fixed-room/:roomId`로 navigate (replace: false)하여 VAD 통역 화면으로 전환
- 접수: `template=reception` + `room` + `kiosk=true` 시 접수용 QR 화면 (동작은 HospitalApp 내부 구현 참조)

### 4.5 /hospital/join/:department (HospitalPatientJoin)

- **department:** `reception` | `consultation` 등 (useParams)
- **쿼리:** `token`, `org`, `room`, `pt` (consultation 시 room 필수, pt는 기존 PT 방 ID)
- **처리:** 언어 선택 후 "통역 시작" 클릭 → POST `/api/hospital/patient` (환자 등록) → POST `/api/hospital/join` (body에 department, patientToken, language, room(consultation일 때), pt 등) → 응답 roomId/sessionId로 navigate
  - **consultation + room:** `/fixed-room/:roomId` (state: fromLang, roleHint guest, inputMode vad, hospitalTemplate consultation 등)
  - **reception:** `/room/:roomId` (ChatScreen)

### 4.6 /fixed-room/:roomId (FixedRoomVAD)

- **역할:** roomId(PT-XXXXXX 등) 고정룸에서 VAD/PTT 통역
- **단계:** waiting → ready → interpreting → ended
- **입력 모드:** inputMode `vad` | `ptt`
  - **vad:** InterpretingVAD 마운트, useVADPipeline만 사용 (로딩 완료 후 vad.start())
  - **ptt:** InterpretingPTT 마운트, MediaRecorder로 녹음 → PCM16 → stt:open/audio/segment_end 전송 (VAD 미사용)
- **소켓:** join(roomId, fromLang, participantId, role, roleHint, siteContext, inputMode 등), fixed-room:start/end, receive-message, receive-message-stream 등

### 4.7 QR 코드 생성

- **HospitalDashboard:** `buildRoomUrl(room, kiosk)` → `/hospital?template=&room=&org=&kiosk=true` 여부
- **buildPatientJoinUrl(room):** 환자가 스캔할 URL → 접수: `/hospital/join/reception?org=`, 상담실: `/hospital/join/consultation?room=&org=`
- **ConsultationKioskView (HospitalApp):** qrUrl = origin + `/hospital/join/consultation?room=...` 또는 reception
- **QR 인쇄:** `https://api.qrserver.com/v1/create-qr-code/?size=280x280&data=` + encodeURIComponent(qrUrl)
- **태블릿용 링크:** HospitalDashboard 방 카드에 "태블릿용 링크 복사" 버튼으로 kiosk=true URL 복사 (구현됨)

## 5. 방(Room) 관리

**관련 파일:** `server.js` (ROOMS Map, create-room, join, leave-room, DB 복원)

### 5.1 ROOMS Map 구조

- **키:** roomId (문자열)
- **값 (meta):** roomType, ownerLang, guestLang, siteContext, locked, ownerPid, participants (participantId → { socketId, lang, displayName, ... }), callSignCounters, expiresAt, hospitalMode, department, patientToken, hospitalSessionId, contextInject 등
- PT- 방은 hospital_sessions와 연동, 서버 재시작 후 join/stt:open 시 DB에서 세션 조회해 ROOMS 재생성 가능

### 5.2 DB 복원

- **hospital_sessions:** room_id(PT-XXXXXX), status, patient_token, dept, assigned_room 등
- join 또는 stt:open 시 `!ROOMS.has(roomId) && roomId.startsWith('PT-')` 이면 DB에서 hospital_sessions 조회 후 ROOMS.set(roomId, meta) 수행

### 5.3 roomId 생성 방식

- **일반:** create-room 시 클라이언트 또는 서버에서 uuid/roomId 생성
- **병원 PT-:** POST /api/hospital/join (consultation 또는 reception) 내부에서 `PT-` + 랜덤 6자(영대문자+숫자) 생성, hospital_sessions에 없을 때까지 재시도

### 5.4 방 생성/삭제/참여/퇴장

- **생성:** `create-room` (일반), `/api/hospital/join` (병원), `create-1to1` (1:1)
- **참여:** `join` (roomId, participantId, fromLang, role, roleHint, siteContext 등) → socket.join(roomId), meta.participants 갱신, partner-joined 등 broadcast
- **퇴장:** `leave-room` (roomId, participantId, reason) → socket.leave(roomId), participants에서 제거, host-left/room-closed 등
- **삭제:** `delete-room` → ROOMS.delete(roomId), room-deleted broadcast
